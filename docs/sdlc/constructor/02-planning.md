# Planning: System Design Constructor

## 1. Scope

### Milestone 1: Данные и типы
- Типы данных (Scenario, Step, Decision, Option, Impact, ReferenceSolution)
- Утилиты для работы со сценариями (получение, фильтрация)
- 2 сценария для MVP: URL Shortener (middle), Chat Messenger (senior)
- Валидационные тесты данных

### Milestone 2: Core-логика
- Хук `useConstructorSession` — управление состоянием сессии (текущий шаг, выборы, навигация, пересчёт метрик)
- Хук `useConstructorProgress` — сохранение/загрузка прогресса в localStorage

### Milestone 3: UI — конструктор
- Страница каталога `/constructor` — список сценариев с фильтром
- Страница сессии `/constructor/[id]/session` — пошаговый конструктор
- Компоненты: StepCard, OptionCard, MetricsRadar, StepNavigator

### Milestone 4: UI — итоговый экран
- Компоненты: TradeOffMatrix, SolutionComparison, ScoreCard
- Интеграция итогового экрана в сессию (последний шаг)

### Milestone 5: Оставшиеся сценарии
- Добавление 6 оставшихся сценариев (Rate Limiter, Notification System, News Feed, Video Streaming, Collaborative Editor, Ride Sharing)

### Milestone 6: Полировка и тесты
- E2E-тестирование полных сценариев
- Адаптивная вёрстка (mobile)
- Ссылка на конструктор с главной страницы

## 2. Зависимости

```
M1 (данные) → M2 (логика) → M3 (UI конструктор) → M4 (итоговый экран) → M6 (полировка)
                                                                    ↑
                                                    M5 (сценарии) ──┘
```

- M2 зависит от M1 (хуки работают с типами и данными)
- M3 зависит от M2 (компоненты используют хуки)
- M4 зависит от M3 (итоговый экран — часть сессии)
- M5 может идти параллельно с M4 (добавление данных не зависит от UI итогов)
- M6 зависит от M4 и M5

## 3. Risks

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Объём данных: 8 сценариев × 6 шагов × 3-5 решений — много контента | Высокая | Средний | MVP с 2 сценариями, остальные добавляются итеративно |
| Radar chart на чистом SVG — сложность реализации | Средняя | Низкий | Fallback на таблицу с цветными индикаторами, если SVG затягивает |
| Несбалансированный impact у опций — неочевидные «правильные» ответы | Средняя | Средний | Тщательная калибровка данных, сравнение с реальными кейсами |
| Навигация назад с пересчётом метрик — edge-cases | Низкая | Низкий | Пересчёт всегда идёт от полного набора текущих выборов, не инкрементально |

## 4. Acceptance Criteria

### AC-1: Каталог сценариев
- [ ] Страница `/constructor` отображает карточки всех сценариев
- [ ] Фильтр по сложности работает корректно
- [ ] Пройденные сценарии отмечены с последним score

### AC-2: Пошаговый конструктор
- [ ] Пользователь проходит шаги последовательно
- [ ] На каждом шаге отображаются решения с вариантами (pros/cons/bestWhen)
- [ ] MultiSelect-решения позволяют выбрать несколько опций
- [ ] Подсказки скрыты и раскрываются по клику
- [ ] Прогресс-бар отражает текущий шаг

### AC-3: Метрики
- [ ] Панель метрик обновляется при каждом выборе
- [ ] Возврат на предыдущий шаг и смена выбора корректно пересчитывает метрики
- [ ] Radar chart (или таблица) визуально отражает 5 метрик

### AC-4: Итоговый экран
- [ ] Отображается текстовое описание собранной архитектуры
- [ ] Trade-off матрица показывает сильные и слабые стороны решения
- [ ] Сравнение с эталоном подсвечивает совпадения и расхождения
- [ ] Score рассчитывается на основе близости к эталону
- [ ] Кнопка «Пройти заново» сбрасывает сессию

### AC-5: Прогресс
- [ ] Выборы сохраняются в localStorage
- [ ] При перезагрузке страницы сессия восстанавливается
- [ ] Каталог показывает статус прохождения

### AC-6: Качество
- [ ] Unit-тесты на хуки и утилиты
- [ ] Валидационные тесты на структуру данных сценариев
- [ ] Корректное отображение на mobile (≥375px)
- [ ] Тёмная тема работает на всех новых компонентах
